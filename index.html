Git and Bash Commands

    Bash Commands

  • pwd: shows current working directory
  • ls: lists all files and folders in current directory
  • cd [enter directory]: navigates between directorys in the path (foward)
  • cd ..: navigates between directorys in the path (backwards)
  • git clone [paste url of repo you want to clone]: clones a repo from github to the current directory
  • mkdir [enter directory name]: creates a new directory
  • touch [enter file name]: creates a new file
  • start [enter file name]: opens file
  • code .: opens VSCode
  • mv [current file location] [new file location]: moves a file to a new location
  • cp [current file location] [copied file location]: copys a file to a new location
  • How to paste in Git Bash: copy what you want to paste and right click to paste
  • code ~/.ssh/ : opens VSCode with all public and private ssh keys

    • Staging and Changing Branches

  • git status: checks what branch you are in
  • git add [file]: adds changes for next commits (stage)
  • git reset [file]: unstages a file while retaining the changes in the working directory
  • git commit -m [write changes here]: commits changes to main
  • git reset: unadds changes in commits while keeping changes in working directory (resets stage)
  • git diff: difference of what is changed but not staged
  • git diff --staged: difference of what is staged but not yet committed
  • git checkout [branch-name]: checks if in specific branch, switches to if not
  • git checkout -b [branch-name]: creates a new branch and switches to it

    • Updating and Merging

  • git branch: lists all branches. * appears next to current active branch
  • git push [alias] [branch-name]: transmits local branch commits to the remote repository branch
  • git pull [alias] [branch-name]: fetch and merge any commits from the tracking remote branch
  • git merge [branch-name]: merges the specific branch into the current one

    • Inspecting and Comparing

  • git log: shows all commits in the current branch's history (type q to exit)
  • git log [branch2]..[branch1]: shows the commits on branch 1 that are not on branch 2
  • git diff [branch2]...[branch1]: shows the difference of what is in branch1 that is not in branch2

HTML

  • DOCTYPE html: declaration that tells the browser what doc type it should expect to open
  • head: contains information about the webpage that the end user will not see
  • meta: defines metadata such as character set, descriptoin, keywords, author, and viewport
  • link: connects an external resource to the HTML doc
  • base: connects an external resource to the HTML doc
  • title: defines the title for the webpage
  • body: represents the visible content shown to the user
  • style: defines the CSS styles associated with a website
  • script: embeds client-side scripts in the HTML doc

CSS: External Style Sheet

  • margin: indicates how much space we want around the outside of an element
  • padding: indicates how much space we want around the content inside an element
  • class attribute (ex. '.card'): allows us to share a CSS rule by assigning a class to an element

JavaScript

  • Variable (var): a named container that alows us to store data in our code
  • Control Flow: the order in which a computer executes code in a script
  • console.log: output a message to the web console by adding an argument
  • 'if': check if the conditions that we provide are true
  • 'else if': if you have more than one condition you use 'else if' for all checks after the first
  • array: a single variable that is used to hold a group a data
  • Numbering an Array: arrays start at [0] so for example, for the array ['square', 'circle', triangle', 'pentagon'], 'square' is at spot [0] and 'pentagon' is at spot [3]
  • x++: instead of writing x + 1, you can write x++
  • function: written as
    function helloworld() { console.log('Hello World, I am a function!') }
  • Calling a Function: functions themselves do nothing until you call them. To call a function write 'helloworld();' and you will receive 'Hello World, I am a function!'
  • How to open Chrome DevTools: Windows use Ctr+Shift+I, MacOS use Command+Option+I
